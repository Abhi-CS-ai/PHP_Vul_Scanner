Vulnerability Types

SQL Injection

SQL injection occurs when untrusted data is sent to an interpreter as part of a command or query, allowing attackers to execute unintended commands or access unauthorized data.

$query = "SELECT * FROM users WHERE username = '" . $_GET['username'] . "';";

Cross-Site Scripting (XSS)

XSS attacks occur when an application includes untrusted data in a new web page without proper validation, allowing attackers to execute scripts in a victim's browser.

echo "Welcome, " . $_GET['name'];

File Inclusion

File inclusion vulnerabilities allow attackers to include files on a server through the web browser, potentially leading to code execution or information disclosure.

include($_GET['page'] . ".php");

Insecure File Upload

Insecure file upload vulnerabilities allow attackers to upload malicious files to a server, potentially leading to remote code execution.

move_uploaded_file($_FILES['userfile']['tmp_name'], "/var/www/uploads/" . $_FILES['userfile']['name']);

Hardcoded Credentials

Hardcoded credentials in source code can lead to privilege escalation or unauthorized access if the code is exposed.

$password = "admin123";

Insecure Cookie Configuration

Cookies without proper security attributes can be vulnerable to theft or manipulation.

setcookie("auth", $token);
